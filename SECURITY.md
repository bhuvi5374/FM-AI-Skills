# Security Policy

## Supported Versions

This repository contains documentation and journey files for CloudBees Feature Management. As this is a public documentation repository, we maintain security for the latest version only.

| Version | Supported          |
| ------- | ------------------ |
| Latest (main branch) | :white_check_mark: |
| Older commits | :x: |

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please report it responsibly:

### For Security Issues:

**DO NOT** create a public GitHub issue for security vulnerabilities.

Instead, please report security concerns to:
- **Email:** bhuvanesh.g5374@gmail.com
- **Subject:** [SECURITY] FM-AI-Skills - Brief Description

### What to Include in Your Report:

1. **Description** - Clear description of the vulnerability
2. **Impact** - What could an attacker do with this vulnerability?
3. **Steps to Reproduce** - Detailed steps to reproduce the issue
4. **Suggested Fix** - If you have ideas for fixing it (optional)

### What to Expect:

- **Acknowledgment:** Within 48 hours
- **Initial Assessment:** Within 7 days
- **Regular Updates:** Every 7-14 days until resolved
- **Resolution:** We aim to resolve critical issues within 30 days

### Public Disclosure:

- We will coordinate with you on public disclosure timing
- Security fixes will be released before public disclosure
- You will be credited (unless you prefer to remain anonymous)

## Security Best Practices for Users

Since this is a **public documentation repository**, please:

### ✅ DO:
- Clone and use the journey files freely
- Share the repository link with colleagues
- Fork the repository for your own use
- Submit Pull Requests with improvements

### ❌ DO NOT:
- Commit any credentials, API keys, or secrets
- Include customer-specific sensitive information
- Upload proprietary CloudBees internal materials
- Share private CloudBees infrastructure details

## Sensitive Information

This repository should contain **ONLY**:
- Generic feature management journey documentation
- Public-facing skill descriptions
- Example workflows and best practices
- Educational content

This repository should **NEVER** contain:
- API keys or credentials
- Customer data or PII
- Internal CloudBees systems information
- Production configuration details
- Secrets or passwords

## Repository Security Features

This repository has the following security measures enabled:

- ✅ Branch protection on `main` branch
- ✅ Required pull request reviews for main branch
- ✅ Security policy (this file)
- ✅ Public repository (no confidential information stored)
- ✅ Regular security audits
